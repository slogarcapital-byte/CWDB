# Session 2026-06-18-001

## Context
A cwdb-ceo-operator request: Jim asked to brainstorm an easy way to pre-fill HubSpot contact info into the first four fields (Name / Address / Phone / Email) of the CWDB deck estimator, so he stops hopping back and forth between the HubSpot contact record and the estimator while running walk-through quotes. Open to either a HubSpot-side button or a Streamlit-side mechanism, whichever is easier. ## Decisions Made
- **Entry point:** in-app "Load customer from HubSpot" picker inside the estimator, NOT a HubSpot-side button. A native button on the HubSpot contact record requires an Enterprise-tier UI-extension app and cannot be built via MCP, so it was ruled out on the Starter tier.
- **Access control:** a passcode gate on the estimator (shared `app_passcode` secret, entered once per session), because the app now reads customer PII (names/addresses/phones/emails) and lives on a public Streamlit URL. If the secret is unset the gate warns and allows, so a missing secret never locks out the live app; the picker also self-disables without its HubSpot token.
- **Deferred (YAGNI):** the same `hubspot_client.py` could later back a `?contact_id=` query-param launch (a bookmarklet "button" from HubSpot) with zero rework; not built because the in-app picker fully solves the stated problem.
