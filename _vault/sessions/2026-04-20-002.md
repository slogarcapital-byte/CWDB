### gallery-featured 3-up grid (task #6)
- Agent `a2d9091e4c74c6272` diagnosed why the CMS Collection List rendered vertically despite `.gallery-featured__grid` having correct grid CSS: Webflow's `DynamoWrapper` (`.w-dyn-list`) + `DynamoList` (`.w-dyn-items`) siblings sit between the grid parent and the items, intercepting the layout. Fix was `display: contents` on both intervening wrappers via two new classes (`gallery-featured__list-wrapper` + `gallery-featured__list`), making the Collection Items bubble up as direct grid children of `.gallery-featured__grid`.
- Breakpoints: desktop 3-col, tablet ≤991px 2-col, mobile ≤767px 1-col.

## Decisions Made

- **hero-interior "blocker" was misdiagnosed.** Jim's `#jim` note reported no image element to bind bg_image to, but the element already existed. The real issue was a missing style class. Logged as a small lesson on inspecting actual state before writing architectural briefs.

- **Phase 2.5 Part B restructured on the fly** from a single destructive strip to a 3-step sequence (visibility → titles → strip) after agent correctly refused to ship. Reduced blast radius from "potentially 8 headless pages" to "cosmetic cleanup of hidden elements."
- **Privacy/Terms title strategy:** hero-interior_about + manual text overrides (Jim's 2 min) chosen over refactoring to CMS (~30 min). Pragmatic for 2-page edge case; CMS refactor could be revisited if more legal pages get added.
- **Master strip + Privacy subheadline fix dispatched in parallel** — independent workflows, can proceed concurrently.
- **Three-revision memory cascade** on Webflow MCP override constraints. Final correct finding: ALL per-instance Component Property overrides are Designer-only on single-locale Webflow sites (Data API requires secondary locale regardless of prop type; Designer MCP exposes no write action). Two intermediate memory drafts were wrong — preserved the correction arc in the final memory for future reference.
