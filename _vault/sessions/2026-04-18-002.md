# Session 2026-04-18-002

## Context
Jim raised that his Obsidian Vault — intended as a full RAG system mirroring the CWDB project (agents, Claude memory, department files) — didn't seem to be functioning as intended. Key folders looked missing: no `.claude/` folder, no agent prompts, no Claude memory content. This session was a diagnostic pass to understand the current vault architecture, explain why it looks broken, and produce a plan to make it actually function as the RAG mirror Jim envisioned.

## Work Done
- **Mapped the current vault state.** Vault root is the entire `CWDB/` directory (`.obsidian/` sits at the root). `_vault/` is a curated knowledge graph with ~50 notes across `agents/` (11 stubs), `sessions/`, `decisions/` (7), `entities/` (1), `phases/` (4), `people/` (3), `markets/` (11), `platforms/` (8), `products/` (1), `canvases/` (5). `.claude/agents/` holds 12 real agent prompt files. Global Claude memory lives at `~/.claude/projects/C--.../memory/` with MEMORY.md, project-state.md, llc-formation.md, feedback-webflow-*.md, phase-f-ids.md, business-contact.md, user_name.md.
- **Identified four root causes** for the "broken vault" perception:
  1. Obsidian hides dot-folders by default — `.claude/` is invisible in the file explorer even though it's inside the vault.
  2. `_vault/agents/*.md` stubs are pointers (path strings), not content mirrors — they never embedded the real prompt.
  3. Global Claude memory lives outside the vault entirely (separate dir under `~/.claude/projects/`).
  4. No sync mechanism — when agent prompts or memory change, the vault index doesn't auto-update.
- **Wrote plan file** `C:/Users/jslog/.claude/plans/delightful-purring-puffin.md` covering six steps: unhide `.claude/` in Obsidian, add `![[...]]` transclusions to all 11 agent notes, create `_vault/claude-memory/` (symlink or sync), create `_vault/departments/` hub notes, leave the working SessionStart hook alone, and optionally build a `/vault-sync` slash command for future drift prevention.
- **Approved via ExitPlanMode.** Plan accepted by user.
- **Read all 11 agent stubs** to prepare transclusion edits. Also confirmed `.claude/agent-memory/` has 12 subfolders (one per agent, including `cwdb-ceo-operator` which has no vault stub).
- **Paused before bulk edits.** Auto mode exited after plan approval. Surfaced four open decisions to user: (1) use vault-root path syntax for `![[...]]` links, (2) create a missing `CEO Operator Agent.md` stub to match the 12th `.claude/agents/cwdb-ceo-operator.md`, (3) choose junction vs scheduled copy vs skip for global memory linkage (OneDrive risk for symlinks), (4) whether to build 8 department index notes or rely on native file explorer. No code changes made to the vault yet — waiting on Jim's call.

## Decisions Made
- Vault is not broken; it's a knowledge-graph *index* that was never expanded into a content *mirror*. Fix by augmenting, not rebuilding.
- Use Obsidian transclusion (`![[...]]`) to bring source content into vault notes rather than copying files. Preserves single-source-of-truth — `.claude/agents/web-dev.md` stays the authoritative prompt, `_vault/agents/Web Dev Agent.md` just embeds it inline with the surrounding metadata/graph.
- Pause after plan approval rather than proceed on assumptions. Four decisions materially change the implementation (path syntax, CEO stub, symlink strategy, dept index notes) and Jim exited auto mode — surface them for explicit confirmation.
